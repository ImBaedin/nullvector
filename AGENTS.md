# Agent Discoveries

- Convex index names must be 64 characters or fewer. Long descriptive names can fail deployment; prefer compact but still readable index names.
- Colony overflow behavior for this project: transport overflow is retained in a per-resource `overflow` bucket on the colony, and local production for a resource is paused while that resource's overflow is greater than zero.
- In this monorepo, check for and follow package-specific `AGENTS.md` files when working inside that package (for example, `packages/backend/AGENTS.md`).
- MVP upgrade timing rule: upgrades are resolved only during explicit colony sync calls; accrual must be segmented so post-completion offline time uses upgraded production rates.
- For Better Auth + Convex web auth, do not manually call `/api/auth/convex/token` from app code; rely on `ConvexBetterAuthProvider` + `convexClient()` plugin and gate Convex queries/mutations with `useConvexAuth()`.
- Queueing model has moved to lane-based `colonyQueueItems`; building upgrades snapshot `fromLevel/toLevel`, cost, and timing at enqueue (cost deducted immediately), and colony/UI queue timing should be driven by `queues.nextEventAt` instead of legacy `activeUpgrade`.
- Shared gameplay DTO/key types now live in `packages/game-logic/src/gameplay.ts` (exported via `@nullvector/game-logic`); prefer importing `BuildingKey`/resource-card queue types from there instead of redefining them in web/backend modules.
- `colonyQueueItems.payload` now includes multiple payload shapes (`buildingUpgrade`, `facilityUpgrade`, and `shipBuild`); in web/backend code, always narrow by `kind` (or payload shape guard) before reading payload-specific fields.
- Product direction clarification: default facility definitions are placeholders; keep unlock logic in place but avoid hard-gating shipyard until non-shipyard facilities are implemented.
- Ship build completion is materialized by `settleShipyardQueue` (typically via colony sync/enqueue paths); `convex/fleet.ts` mutations do not currently run that settlement, so fleet actions can see stale ship counts until a sync occurs.
- Shipyard wiring decisions (2026-03-05): implement ship queue cancellation with full refunds, keep unlock checks shipyard-level-only for now (no research gating yet), and set ship build quantity UI cap to 10,000 to match backend.
- Overflow reconciliation rule: `settleColonyAndPersist` must persist `colony.overflow`, and sync should move overflow back into `resources` whenever storage headroom exists; otherwise production can stay paused with stale overflow.
- Fleet page implementation scope (2026-03-08): `/game/colony/$colonyId/fleet` should be colony-scoped (active colony only), not empire-wide.
- Fleet transport targeting policy (2026-03-08): transport missions may target other players' colonies when inbound policy allows; own-colony-only is not required.
- Fleet destination UX decision (2026-03-08): fleet planner destination selection should include Star Map picking (not coordinates-only fallback).
- Fleet sync/UI direction (2026-03-08): colony layout timed sync should use colony-relevant fleet events (outgoing + incoming) rather than player-wide-only timestamps; fleet route active expeditions should include incoming operations for the viewed colony.
- Repo tooling note (2026-03-10): for formatting/linting in this monorepo, use the root commands `bun run format` and `bun run lint:fix` rather than invoking package-local format/lint scripts directly.
- Server-owned timing architecture (2026-03-10): queue and fleet due-event resolution is now scheduled in Convex via internal `scheduler` functions; client `useGameTimedSync`/visibility sync was removed, schedule metadata lives on `colonies` and `fleetOperations`, and `syncColony` is no longer the normal completion path.
- Worldgen caveat (2026-03-11): the current generator pipeline only creates sectors with `sectorType: "core"`; `frontier` exists in schema/query types but is not generated yet, so any hostile/frontier feature work needs either a generator expansion or a compatibility fallback for existing universes.
- Contract tuning note (2026-03-11): low combat budgets should not force-spawn higher-tier enemy ships; in `packages/game-logic/src/contracts.ts`, fleet budget distribution now skips ship classes that the remaining budget cannot support instead of guaranteeing at least one unit.
- Universe object naming decision (2026-03-11): persist generated names on `galaxies`/`sectors`/`systems`/`planets`; add `name` fields as optional for rollout, backfill with a Convex migration, and keep query-time generated fallback until migration completes.
- Fleet operation architecture note (2026-03-11): `fleetOperations.kind`/schema includes `combat`, but `packages/backend/runtime/gameplay/fleetV2.ts:createOperation` currently only accepts `transport` and `colonize`; contract combat uses a separate acceptance flow and `settleContractAtTarget`.
- Colony defense persistence gap (2026-03-11): shared game logic has `DefenseCounts`/combat support, but owned-colony state/schema currently has no `colonyDefenses` companion row or defense queue payload/lane, so defenses are not yet first-class colony state like buildings or ships.
- Colony raid design clarification (2026-03-11): sectors are only colonizable once cleared, so nearest hostile planets should never be treated as same-sector raid sources; NPC raids should not begin until the defending player reaches rank 5.
- Colony raid outcome clarification (2026-03-11): successful defenses should grant salvage based on destroyed attacker ships, while failed defenses should reduce the defending player's rank XP.
- Notification system status (2026-03-11): backend already persists mission/raid outcome data in `fleetOperationResults`, `fleetEvents`, `contractResults`, and `npcRaidResults`, but there is no first-class notifications table/query layer yet; the web notifications modal and settings are currently mock/local-state only.
- Convex paginated feed note (2026-03-12): web `usePaginatedQuery` expects the standard Convex pagination result shape (`page`, `continueCursor`, `isDone`); custom wrappers like `items`/`nextCursor` will typecheck poorly and break the paginated hook contract.
- Fleet visibility rule (2026-03-12): colony fleet views should merge owner-scoped operations with target-colony-scoped inbound operations; recipient-visible incoming transports rely on the `fleetOperations.by_tcol_st_evt` index rather than owner-only queries.
- Contracts architecture (2026-03-14): available contract offers are no longer persisted as `contracts` rows; nearby hostile targets are projected per colony in `colonyContractCandidates`, per-planet slot sequencing lives in `contractBoardState`, accepted/resolved missions still use `contracts`, and available offers no longer expire on a timer.
- Raid scheduling architecture (2026-03-14, updated 2026-03-20): colony raids are cron-reconciled from `colonies.nextNpcRaidAt`; use the `by_next_npc_raid_at` index for due spawn sweeps and `npcRaidOperations.by_status_event` for due raid resolution, rather than relying on full-colony reconciliation or per-raid scheduled jobs.
- NPC raid direction clarification (2026-03-20): raids no longer require nearby hostile sectors or planets; they are scheduled per colony from an unknown origin, with deterministic faction/timing derived from colony state rather than local hostility data.
